# RohitOS Agent Rules
1. STABILITY FIRST: Do not rewrite the whole project. No architecture destruction.
2. STRICT MODULARITY: Commands stay in `commands/`, voice stays in `voice/`. Do not tangle dependencies.
3. SMALL SCOPED CHANGES: Modify the minimal files possible. Prefer adding isolated modules over rewriting systems.
4. TOKEN EFFICIENCY: Minimize token usage. Avoid rewriting unchanged code.
5. FILE SAFETY: Only modify explicitly requested files.
6. TESTING: Preserve existing working functionality.
7. READABILITY: Keep code beginner-friendly and modular.
8. WORKFLOW: Build one stable feature at a time.
